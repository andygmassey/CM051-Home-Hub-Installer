## Findings (3)

### 1 · P2 · Untracked background Ollama process

**File:** `install.sh:1273`

```bash
ollama serve &>/dev/null &
```

Spawns Ollama daemon in the background but if the later
`open -gj -a Ollama` succeeds, the backgrounded process isn't
tracked for cleanup. If the script exits early (trap, Ctrl-C) it
leaves an orphaned server. A `wait` or `trap` entry would make
the cleanup explicit.

**Category:** 3.

### 2 · P3 · Empty if-block with `:` no-op

**File:** `install.sh:138`

```bash
if [ -n "$LIFELINE_BETA" ]; then
  : # Beta mode is default for now
fi
```

Stub from the HR015 split. Adds no logic, just a placeholder
comment. Either flesh out the licensing check or remove the
block.

**Category:** 2.

### 3 · P3 · `mktemp` file not cleaned on `mv` failure

**File:** `install.sh:2104-2106`

```bash
TMP_ENV=$(mktemp)
...
mv "$TMP_ENV" "$ENV_FILE"
```

If `mv` fails (disk full, permissions), `$TMP_ENV` is orphaned in
`/tmp`. Wrap in `trap 'rm -f "$TMP_ENV"' EXIT`, or rewrite as
in-place sed.

**Category:** 3.
